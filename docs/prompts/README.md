# Prompt Library

Store reusable frontend prompts and approved planning artifacts here.

Use `docs/project/PROMPT_TEMPLATE.md` as the baseline template for new frontend planning prompts.

Prompt files should document:

- Goal.
- Backend context.
- Current frontend context.
- Scope.
- Out of scope.
- Security rules.
- Verification commands.
- Approval status.

Do not store secrets, tokens, credentials, or private backend values in prompts.
